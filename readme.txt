1 Nginx
Nginx는 react의 빌드 파일들로 연결 해주는 역할

2 Front(React)
    1. `App.js`
    - 역할: 애플리케이션의 메인 컴포넌트
    - 주요 기능:
     - 전체 레이아웃 관리
     - 인증 상태 관리 (로그인/로그아웃)
     - 테마 전환 (다크 모드/라이트 모드)
     - WebSocket 연결 관리 (useWebSocket 훅 사용)

    2. `ChatPage.js`
    - 역할: 채팅 페이지 컴포넌트
    - 주요 기능:
     - 채팅 메시지 표시
     - 메시지 입력 및 전송 관리
     - WebSocket을 통한 실시간 메시지 처리

    3. `AuthModal.js`
    - 역할: 인증 모달 컴포넌트
    - 주요 기능:
     - 로그인 및 회원가입 폼 제공
     - 사용자 인증 처리
     - 입력 유효성 검사

    4. `ChatMessages.js`
    - 역할: 채팅 메시지 표시 컴포넌트
    - 주요 기능:
     - 메시지 목록 렌더링
     - 메시지 스크롤 관리
     - 사용자별 메시지 스타일 적용

    5. `ChatInput.js`
    - 역할: 채팅 입력 컴포넌트
    - 주요 기능:
     - 메시지 입력 폼 제공
     - 메시지 전송 처리
     - 채팅 금지 상태 관리

    6. `useWebSocket.js` (hooks 폴더 내)
    - 역할: WebSocket 관리를 위한 커스텼 React 훅
    - 주요 기능:
     - WebSocket 연결 설정 및 관리
     - 메시지 송수신 처리
     - 연결 상태 및 오류 처리

    7. `ThemeContext.js` (contexts 폴더 내)
    - 역할: 테마 관리를 위한 React Context
    - 주요 기능:
     - 다크 모드/라이트 모드 상태 관리
     - 테마 전환 기능 제공

    ## 스타일

    - `base.css`: 기본 스타일 정의
    - `components.css`: 컴포넌트별 스타일 정의
    - `utilities.css`: 유틸리티 클래스 정의
    - `App.css`: App 컴포넌트 관련 스타일
    - `AuthModal.css`: 인증 모달 관련 스타일
    
3 Backend(Fastapi)
    1. `main.py`
    - 역할: FastAPI 애플리케이션의 주 진입점
    - 주요 기능:
     - FastAPI 애플리케이션 및 라우트 정의
     - CORS 미들웨어 설정
     - WebSocket 연결 처리
     - 사용자 인증 (로그인/회원가입) 엔드포인트
     - 최근 메시지 조회 엔드포인트

    2. `connection_manager.py`
    - 역할: WebSocket 연결 및 실시간 메시지 브로드캐스팅 관리
    - 주요 기능:
     - 활성 WebSocket 연결 관리
     - 메시지 브로드캐스팅
     - 스팸 방지 및 사용자 차단 로직
     - 연결된 사용자 수 추적

    3. `postgresql_manager.py`
    - 역할: PostgreSQL 데이터베이스 연결 및 쿼리 실행
    - 주요 기능:
     - 데이터베이스 연결 풀 관리
     - 사용자 등록 및 로그인 처리
     - 메시지 저장 및 조회
     - 사용자 정보 조회

    4. `background_tasks.py`
    - 역할: 비동기 백그라운드 작업 관리
    - 주요 기능:
     - 주기적인 사용자 수 업데이트
     - 오래된 데이터 정리
     - 연결 상태 확인
